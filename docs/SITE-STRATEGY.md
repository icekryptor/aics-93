## 1. Позиционирование и язык общения с клиентом (ToV)

### Формула позиционирования
**«Один инженер + ИИ-конвейер = скорость и цена, недоступные агентству. Студийное качество,
инженерная честность».** Ниша «индивидуальный сайт за 7–14 дней» между конвейером-шаблонкой
(3 дня) и бутиком (30–45 дней) пуста — занимаем её явно и повсюду.

### Правила языка (дополняют «голос» в DESIGN-SYSTEM.md §10)

1. **Цифры вместо эпитетов.** Не «качественно и быстро», а «7–14 дней, до 70% экономии,
   логистика 7–10%». Метрики-крючки в подзаголовках — паттерн топ-выдачи.
2. **Честность как приём** (эталон — aify.studio): говорить, где ИИ не нужен, что не входит
   в цену, когда лучше конструктор. Анти-хайп продаёт лучше хайпа в 2026.
3. **От первого лица.** «Я собираю / я вернусь с расчётом» — личный бренд оператора работает
   (Коган), обезличенное «мы» у соло-студии звучит фальшиво. Уже частично так — закрепить.
4. **Показывать механику, не магию.** Не «нейросети создадут сайт», а «агенты компилируют
   данные брифа → дизайн-система генерирует блоки → я проверяю каждую строку». Отстройка
   и от агентств, и от нейро-конструкторов.
5. **Обучающие сравнения** (паттерн «Имя.»): блоки «ПЛОХО / ХОРОШО», «конструктор / ИИ-движок» —
   уже есть comparison-таблицы, расширять на формулировки офферов.
6. **Лексика сегмента МСБ**: без «диджитал-экосистем» и «синергий»; слова клиента — заявки,
   маржа, окупаемость, «сколько стоит», «когда будет готово».
7. **Запретные штампы**: «индивидуальный подход», «команда профессионалов», «под любой бюджет»,
   «качество и надёжность» — маркеры шаблонного рынка.

## 2. Структура сайта — чего не хватает

| Пункт | Зачем (обоснование из разведки) | Усилие |
|---|---|---|
| **Страница/блок «Как работает ИИ-конвейер»** | Белое пятно №2: никто не объясняет механику скорости. Наш главный дифференциатор без витрины | M |
| **Публичные пакеты: цена + срок + состав** | Комбинация «фикс-цена + метрики» свободна (пятно №4). Формула карточки (Repina+Depot): цена «от» → состав списком → срок в днях → что включено → от чего растёт цена. Ценовой разрыв рынка фирстиля: фрилансер 30–50 т.р. ↔ агентство от 600 т.р. — наша полка **150–300 т.р. / 10–14 дней**. Формат «3 пакета в таблице сравнения» на рынке брендинга МСБ отсутствует — занять первым | M |
| **«Дизайн-система, которая живёт в коде»** как отдельный оффер фирстиля | Никто из топ-6 брендинга не продаёт дизайн-системы МСБ (Repina — от 660 т.р.); у нас токены/UI-кит/компоненты — часть конвейера. Уникальное техпреимущество | S (упаковка текстом) |
| **Входной продукт: ИИ-аудит за фикс** | 15–30 т.р. / 3 дня — верх воронки перед фирстилем/сайтом (паттерн ONY/Depot «экспресс-аудит») | S–M |
| **Кейсы с метриками «Было → Стало → Результат»** | Пятно №4; формат aify. Кейс 01 дополнить цифрами, кейсы 02–03 сразу строить по формату | M |
| **Страница «Оператор» (о студии/личный бренд)** | Пятно №5: фото, история, процесс, публичность; связка с @aics93 | S |
| **Низкопороговый вход** — «прототип за 24 часа» / бесплатный ИИ-аудит / «напишите „хочу структуру“» | Приём «Имя.» + aify; квизы уже есть, нужен ещё более лёгкий шаг | S |
| **Соцдоказательства**: отзывы, логотипы клиентов | Гигиенический минимум рынка, у нас ноль | S–M |
| **Экспресс-продукт по формуле Лебедева** | «Сайт-экспресс за 7 дней, фикс-цена, 2 итерации включены, N слотов в месяц» + витрина продукта. «Сайт-экспресс» как продукта нет даже у Лебедева — ниша пуста | M |
| **Лестница цен** | Дешёвый ИИ-вход (аудит/концепт) → экспресс-сайт → дизайн-система/сервис; нижний этаж маркетирует верхний | зависит от пакетов |
| **Подписка «поддержка/мес»** отдельной страницей | Паттерн «Очень»: retainer = рекуррентная выручка + якорь для МСБ, боящегося больших проектов | S–M |
| **Счётчики-доказательства на первом экране** | Вместо наград: запущено N, медианный срок X дней, PageSpeed 95+ (паттерн Red Collar) | S |
| **SLA ответа + «что будет дальше»** | «Отвечу за 24 часа» + 3–5 шагов после заявки (паттерн AGIMA); соло даёт SLA агрессивнее интегратора | S |
| **GEO-страница «Hey AI, learn about us»** | Паттерн AGIMA: самоописание для LLM-краулеров; для ИИ-студии — ещё и позиционирование. llms.txt уже есть — расширить до страницы | S |
| **Хаб-страницы ниш** (бэклог, по одной по мере готовности) | Отраслевые хвосты свободны; шаблон «сайт для ‹ниша› — цена+срок» | M каждая |

## 3. Дизайн (в рамках существующей ДС)

- Инженерная эстетика — уже главная визуальная отстройка сегмента (пятно №3). Не разбавлять.
- Добавить «артефакты процесса» как дизайн-контент: скрин harness-панели, фрагменты пайплайна,
  терминальные логи — там, где появится блок про ИИ-конвейер.
- Карточкам кейсов — слот под метрики (значение + подпись, паттерн hero.stats уже есть).
- Фото оператора (для страницы «Оператор» и, возможно, футера) — единственный тип контента,
  которого ДС пока не описывает; при внедрении добавить правило в DESIGN-SYSTEM.md.

## 6. Приоритизация

### Quick wins (сделано в этом заходе)
- ✅ Публикация /solutions: индекс + шапка + тизер главной + футер + sitemap
- ✅ SEO-мета по ядру: главная/OG/keywords, услуги, решения; фикс title-шаблона
- ✅ docs: COMPETITORS.md, SEO-CORE.md, SITE-STRATEGY.md

### Ближайшие (S, по 0,5–1 дню)
1. Цели в Метрике + Вебмастер/GSC (частично руками юзера)
2. Низкопороговый вход: строка «напишите в TG „хочу разбор“ — вернусь со структурой и вилкой
   цены» рядом с CTA + SLA «отвечу за 24 часа» + шаги «что будет дальше»
3. Страница «Оператор» (нужны: фото, 3–4 факта, история)
4. Счётчики-доказательства на первом экране (запущено N, медианный срок, PageSpeed)
5. GEO-страница «Hey AI, learn about us» (расширение llms.txt)

### Средние (M, 1–3 дня каждая)
4. Блок/страница «Как работает ИИ-конвейер» + артефакты процесса
5. Публичные пакеты с фикс-ценой (нужно решение юзера по ценам!)
6. Кейс 01 — дометрики; кейсы 02–03 по формату «Было → Стало → Результат»
